# Session Notes - 2026-04-24

## 1. Remotion / Preview / Export parity

### Kết luận ngắn

Logic vừa sửa là **đúng với nguyên tắc của Remotion**, không phải workaround trái quy luật.

### Vì sao đúng

Trong Remotion, thời gian của một clip nên được đặt bằng:

- `Sequence from={clip.from} durationInFrames={clip.durationInFrames}`
- media bên trong clip dùng local frame của `Sequence`

Nếu clip nằm ở `02:24` trên timeline mà component `Video` / `Audio` không nằm trong `Sequence`, nó sẽ vẫn đọc theo frame toàn cục của composition. Khi đó:

- timeline nhìn như clip bắt đầu ở `02:24`
- nhưng media source thực tế đã bị tua theo frame toàn cục
- tới đúng độ dài asset thì nó sẽ hết source và ra đen / im lặng

Đó là bug vừa gặp.

### Điều này có an toàn cho export không

Có, **nếu export dùng đúng cùng project model và cùng composition logic như Preview**.

Đây là hướng đúng cần giữ:

1. `project` là source of truth
2. `EditorPreviewComposition` render từ `project`
3. export render cũng dùng chính `project` đó
4. mọi timing của clip đều đi qua `Sequence`

Nếu giữ 4 điểm này thì:

- Preview thấy gì
- Timeline mô tả gì
- Export sẽ ra đúng như vậy

### Khi nào Preview và Export có thể lệch nhau

Sẽ lệch nếu một trong các việc sau xảy ra:

1. Preview dùng một model, export dùng model khác
2. Preview render bằng logic local, export render bằng logic global
3. một số state chỉ tồn tại trong UI/runtime mà không được ghi vào `project`
4. layer order / mute / hide / transform / trim không được serialize đầy đủ

### Với JSON trả về cho BE

Format kiểu này là hoàn toàn làm được:

```ts
const initialData: RemotionProjectJson = {
  fps: 30,
  durationInFrames: 600,
  width: 1280,
  height: 720,
  tracks: [
    {
      id: "track-video",
      name: "Main Video Track",
      clips: [
        {
          id: "intro-video",
          type: "video",
          sequenceProps: {
            from: 0,
            durationInFrames: 150,
          },
          mediaProps: {
            src: "https://www.w3schools.com/html/mov_bbb.mp4",
            style: { width: "100%", height: "100%", objectFit: "cover" },
            muted: true,
          },
          transitionToNext: "fade",
          transitionDuration: 30,
          animations: [
            { type: "blur", direction: "in", durationInFrames: 30, value: 20 },
          ],
        },
      ],
    },
  ],
};
```

### Nhưng nên chốt model theo hướng nào

Nên tách làm 2 lớp:

1. **Editor project model chuẩn nội bộ**
2. **Serializer sang RemotionProjectJson cho BE**

Lý do:

- editor cần model dễ sửa, dễ kéo thả, dễ maintain
- BE / render pipeline cần model ổn định, serializable, rõ timing
- không nên để UI state trộn trực tiếp vào JSON export

### Khuyến nghị model

Nội bộ editor nên tiếp tục giữ dạng đang có:

- `project.video`
- `project.tracks`
- `project.clips`
- `clip.from`
- `clip.durationInFrames`
- `clip.sourceStartFrame`
- `clip.transform`
- `clip.layerIndex`

Sau đó map sang JSON cho BE:

- `clip.from` -> `sequenceProps.from`
- `clip.durationInFrames` -> `sequenceProps.durationInFrames`
- `clip.src`, `volume`, `muted`, `objectFit` -> `mediaProps`
- effect / animation của editor -> `animations`
- transition editor -> `transitionToNext`, `transitionDuration`

### Kết luận kỹ thuật

- Dùng `Sequence` cho clip timing là đúng chuẩn Remotion
- Không ảnh hưởng xấu tới export
- Ngược lại, đây là điều kiện cần để Preview và Export ra giống nhau
- JSON cho BE nên là output serialize từ editor model, không phải source model duy nhất của UI

## 2. Những gì đã làm được trong hôm nay

### Timeline / playback / duration

- Bỏ default duration giả khi project rỗng; empty project về `0`
- Fix duration hiển thị theo end boundary thay vì frame cuối renderable
- Fix seek start/end để timeline scroll đúng về đầu / cuối
- Fix logic playback duration theo clip kết thúc muộn nhất trên timeline
- Fix playhead không bị mất marker head do stack sai
- Fix playhead viewport layer để không bị `scrollY` cùng `TimelineBody`
- Fix playhead không đè lên horizontal scrollbar
- Giảm lag bằng cách tách playhead khỏi re-render của toàn bộ Timeline
- `setTimelineScroll` được sync theo `requestAnimationFrame`

### Timeline ruler / z-index / layout

- Chỉnh lại stack:
  - `track headers` trên cùng trong vùng content
  - `playhead` trên `ruler`
  - `ruler` dưới `playhead`
- Fix `Ruler` không đè lên `track headers`
- Fix lane height theo clip cao nhất trong lane
  - lane có `video` thì lane ăn theo `video`

### Playback / Preview sync

- Sync lại seek playhead với Preview
- Preview nhận event seek trực tiếp để scrub bám timeline tốt hơn
- Khi đang play mà seek playhead thì playback dừng
- Fix preview không tự cộng duration giả khi project rỗng

### Media clip timing / preview correctness

- Fix bug lớn: clip đặt lệch khỏi `00:00` bị đen / kết thúc sai
- Composition giờ render clip trong `Sequence`
- Điều này làm timing của video/audio/image bám đúng `clip.from`

### Upload / media groundwork

- Đã chốt hướng upload media và ghi riêng ở `MEDIA_UPLOAD_PIPELINE_PLAN.md`
- Đã xử lý các vấn đề:
  - clip upload bị thụt khỏi `00:00`
  - media preview fill tốt hơn
  - bỏ hiệu ứng mặc định khi media mới xuất hiện

### Timeline interactions

- Click ruler để seek
- Drag playhead để seek
- Delete clip
- Split clip tại playhead
- Tăng số lượng ticks khi timeline ngắn / ít track
- Xóa clip thì lane controls không còn bị giữ lại sai

### Drag and drop clips / lanes / layering

- Chuyển sang `dnd-kit`
- Clip có thể move giữa các lane
- Có ghost preview khi drag
- Drag vượt trên cùng / dưới cùng có thể tạo lane mới
- Lane rỗng sau khi move clip sẽ bị remove
- Clip mới thêm vào tạo lane mới phía trên
- Layer logic:
  - lane trên che lane dưới trong Preview
  - reorder lane sẽ sync lại `layerIndex`
- Trong cùng lane:
  - clip không được overlap
  - nếu drop đè, clip sẽ nhảy ra sau clip đang chiếm chỗ

### Track controls

- Toggle `mute`
- Toggle `hide`
- Preview đã respect `track.isMuted`
- Preview đã respect `track.isHidden`

### Text editing in Preview

- Add text clip và hiện ngay trên Preview
- Có 3 state:
  - display
  - selected
  - edit mode
- Double click để edit text
- Click để select / move
- Fix caret nhảy đầu dòng
